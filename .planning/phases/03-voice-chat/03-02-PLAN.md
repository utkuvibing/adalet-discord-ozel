**1. src/renderer/components/VoiceControls.tsx** -- Create the Discord-style bottom bar:

**Props interface:**
```typescript
interface VoiceControlsProps {
  myVoiceState: VoiceState;
  onToggleMute: () => void;
  onToggleDeafen: () => void;
  activeRoomId: number | null;
}
```

**Layout:** Fixed at bottom of sidebar. Three icon buttons in a row:
- **Mic icon** (mute toggle): Show a microphone SVG icon. When muted, show a red slash through it and red tint. When unmuted, show green (#7fff00) tint. Click toggles mute via `onToggleMute`.
- **Headphone icon** (deafen toggle): Show a headphones SVG icon. When deafened, show red slash and red tint. When not deafened, show default gray. Click toggles deafen via `onToggleDeafen`. NOTE: When user deafens, auto-mute as well (Discord behavior). When user un-deafens, restore previous mute state.
- **PTT toggle button**: A small text button showing the current mode -- "OPEN MIC" or "PTT: [key]". Click toggles between open mic and PTT mode.

**PTT mode logic:**
- State: `pttEnabled` boolean, `pttKey` string (default: `'Insert'` -- a sensible gaming key that doesn't conflict with common game bindings)
- When PTT mode enabled: call `window.electronAPI.registerPTTShortcut(pttKey)` and listen via `window.electronAPI.onPTTStateChange((pressed) => { ... })`.
  - When pressed=true: unmute mic (call parent's mute control or directly enable local audio track)
  - When pressed=false: mute mic
- When PTT mode disabled: call `window.electronAPI.unregisterPTTShortcut()` and restore open-mic behavior.
- Show a subtle pulsing dot or border glow when PTT key is held and transmitting.

**PTT hotkey rebind:** Add a small "rebind" affordance next to the PTT button. When clicked, show a "Press a key..." prompt and capture the next key press, convert to Electron accelerator format, re-register with the new key.

**Styling:** Dark background matching sidebar (#111111). Border-top: 1px solid #2a2a2a. Compact height (~48px). Icons should be simple SVG paths (mic, headphones) or Unicode symbols as fallback. Green (#7fff00) for active/unmuted states, red (#ff4444) for muted/deafened.

Only render the bar content when `activeRoomId !== null` (user is in a room). When not in a room, show nothing or a minimal "Not in a room" text.
