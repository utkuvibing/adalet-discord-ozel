**Create src/server/upload.ts:**

1. Install `multer` and `@types/multer` as dependencies (add to action that executor must run `npm install multer @types/multer`).

2. Import dependencies: `multer`, `path`, `crypto` from `node:crypto`, `fs` from `node:fs`, Express types, db and schema, and the Socket.IO server type.

3. Configure multer disk storage:
   - Destination: `uploads/` directory in the app's user data folder. Use `app.getPath('userData')` for packaged apps or a local `uploads/` folder for development. Since this runs in the main process context, use `process.cwd() + '/uploads'` for dev (Electron Forge runs from project root). Create the directory if it doesn't exist (`fs.mkdirSync(uploadsDir, { recursive: true })`).
   - Filename: Generate unique filenames using `crypto.randomUUID() + path.extname(originalname)` to prevent collisions and path traversal.
   - File size limit: 25MB (`limits: { fileSize: 25 * 1024 * 1024 }`).

4. Create `registerUploadRoutes(app: Express.Application, io: TypedIO)` function:
   - POST `/upload` route:
     - Use multer single file upload: `upload.single('file')`
     - Validate required fields from request body: `roomId` (number), `userId` (number)
     - Validate the file exists on `req.file`
     - Insert a message into the DB:
       ```typescript
       db.insert(messages).values({
         roomId: parseInt(req.body.roomId),
         userId: parseInt(req.body.userId),
         content: req.body.content || '', // optional text alongside file
         fileUrl: `/uploads/${req.file.filename}`,
         fileName: req.file.originalname,
         fileSize: req.file.size,
         fileMimeType: req.file.mimetype,
       }).run()
       ```
     - Get the inserted message ID
     - Look up the user for displayName and avatarId
     - Construct and emit a ChatMessage to the room via `io.to(ROOM_PREFIX + roomId).emit('chat:message', chatMessage)`
     - Return 200 JSON with the ChatMessage
   - Error handling: Return 400 for missing fields, 413 for file too large (multer error), 500 for server errors

5. Export `registerUploadRoutes` and `getUploadsDir()` (the resolved uploads directory path, for use by static serving).
