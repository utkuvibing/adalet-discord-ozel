**1. Create src/renderer/hooks/useSocket.ts** -- Custom hook managing Socket.IO client connection:

- Import `io, Socket` from `socket.io-client`
- Import `ServerToClientEvents, ClientToServerEvents` from `../../shared/types`
- Type alias: `TypedSocket = Socket<ServerToClientEvents, ClientToServerEvents>`

Export `useSocket()` hook that returns `{ socket, connectionState, error, connect, disconnect }`:

- `connectionState`: `'disconnected' | 'connecting' | 'connected' | 'reconnecting'`
- `error`: `string | null` (error message from server, e.g., "EXPIRED_TOKEN")
- `connect(serverAddress: string, token: string, displayName: string): void` -- Creates socket via `io(\`http://${serverAddress}\`, { transports: ['websocket'], auth: { token, displayName }, reconnection: true, reconnectionAttempts: Infinity, reconnectionDelay: 1000, reconnectionDelayMax: 10000, randomizationFactor: 0.5, timeout: 10000 })`. Force WebSocket transport per research Pitfall 2 (Electron optimization).
- Sets up event listeners: `connect` -> set state 'connected', `disconnect` -> set state 'disconnected', `connect_error` -> parse error message (MISSING_TOKEN, INVALID_TOKEN, EXPIRED_TOKEN, TOKEN_LIMIT_REACHED) and set to `error` state. Map error codes to user-friendly messages per locked decisions:
  - EXPIRED_TOKEN: "This invite has expired. Ask the host for a new invite link."
  - INVALID_TOKEN: "This invite link is not valid. Check for typos or ask the host for a new one."
  - TOKEN_LIMIT_REACHED: "This invite has reached its use limit. Ask the host for a new invite link."
  - MISSING_TOKEN: "No invite token provided."
  - Default: "Can't reach server. Retrying in 5s..."
- For reconnection: listen to `reconnect_attempt` -> set state 'reconnecting', `reconnect` -> set state 'connected'
- `disconnect(): void` -- Calls `socket.disconnect()`, cleans up
- Use `useRef` for socket instance (not state, to avoid re-renders on every emit)
- Use `useCallback` for connect/disconnect to stabilize references
- Cleanup on unmount: disconnect socket

**3. Create src/renderer/hooks/useWebRTC.ts** -- Perfect Negotiation hook managing peer connections:

This hook is called once per room. It manages a `Map<string, RTCPeerConnection>` of peer connections (one per remote peer).

Export `useWebRTC(socket: TypedSocket | null, mySocketId: string | null)` that returns `{ peerConnections, addPeer, removePeer }`:

- `peerConnections`: `Map<string, RTCPeerConnection>` (ref, not state -- connections are mutable)
- Uses `ICE_SERVERS` from `../../shared/iceConfig`

**addPeer(remoteSocketId: string, initiator: boolean):**
- Creates `new RTCPeerConnection({ iceServers: ICE_SERVERS })`
- Determines polite/impolite role: `const polite = mySocketId! < remoteSocketId` (lexicographic comparison per research)
- Implements Perfect Negotiation pattern (from MDN/research):
  - `let makingOffer = false; let ignoreOffer = false; let isSettingRemoteAnswerPending = false;`
  - `pc.onnegotiationneeded`: set makingOffer=true, `await pc.setLocalDescription()`, emit `sdp:offer` with `{ to: remoteSocketId, description: pc.localDescription }`, set makingOffer=false
  - `pc.onicecandidate`: emit `ice:candidate` with `{ to: remoteSocketId, candidate }`
  - ICE candidate queuing (research Pitfall 5): maintain `pendingCandidates: RTCIceCandidateInit[]`. On incoming ICE candidate from this peer, if `pc.remoteDescription` is set, call `pc.addIceCandidate(candidate)`. Otherwise push to queue. After every `setRemoteDescription`, drain the queue.

**Socket event handlers** (set up when socket is available, cleaned up on unmount):
- `sdp:offer` handler: If `from` matches a known peer, apply Perfect Negotiation logic (check readyForOffer, offerCollision, ignoreOffer per polite/impolite role). If from an unknown peer, call `addPeer(from, false)` first.
- `sdp:answer` handler: Set `isSettingRemoteAnswerPending = true`, `await pc.setRemoteDescription(description)`, drain pending ICE candidates.
- `ice:candidate` handler: Queue or add immediately based on whether remoteDescription exists.
- `room:peers` handler: For each peer socket ID received, call `addPeer(peerId, true)` -- the joiner initiates connections to existing peers.

**removePeer(remoteSocketId: string):**
- Calls `pc.close()` on the connection
- Removes from the Map

**Cleanup:** On unmount, close all RTCPeerConnections and clear the Map. Remove all socket event listeners.

**Important:** This hook does NOT add audio tracks yet (that is Phase 3 -- Voice Chat). It only establishes the peer connections and signaling channel. The connections will be "data-only" until Phase 3 adds `getUserMedia` and `addTrack`. The hook should be structured so that Phase 3 can easily call `pc.addTrack(audioTrack, stream)` on existing connections.
